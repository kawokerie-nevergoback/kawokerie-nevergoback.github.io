Notre projet


Le but principal de l'équipe Kawok enjoyers peut être résumé en une phrase : "Innover et progresser en s'amusant". Nous innoverons en créant un jeu original ne copiant pas les jeux dominant le marché aujourd'hui. Malgré le fait que ce type de jeu "Death run" existe déjà, nous allons développer le concept dans un jeu novateur qui en repoussera les limites. En effet, nous mêlerons le Death run à des fonctionnalités exclusives.

Problèmes rencontrés ❌ Solutions apportés 👍
Partage du projet à jour entre les membres (avec Version Control) Migration vers Github en utilisant Github Desktop pour clone / push le projet
Déplacement peu fluide en utilisant l'ancien Input System Utilisation du New Input System
Mouvement de caméra saccadé Implémentation du plugin CineMachine pour une caméra principale fluide
Sensibilité de souris qui varie en fonction de l'amplitude du mouvement de caméra avec le New Input System Utilisation de Input.GetAxis pour les mouvements de caméra
Le joueur tombe en sautant sur une plateforme en mouvement Mise du joueur en tant que child de la plateforme pour qu'il soit fixé dessus
Mauvaise échelle en important une map de Blender sur Unity Utilisation de l'objet Terrain sur Unity
Problème de gravité sur les OS basés sur Linux Retrait du component rigidbody
Bug de gravité des autres joueurs Retrait du component rigidbody
Controle de l'autre joueur avec le multi Désactivation de la caméra et activation au runtime
Probleme d'axe en fonction du parent Tout faire avec le même parent pour ne pas avoir de problèmes de dimensions et d'axes
Map trop sombre Utilisation d'un object lightprobe pour afficher les reflets
Problème d'optimisation avec la pluie Réduction du nombre de particule et de leur durée de vie
Problème d'optimisation avec les arbres Réduction du nombre d'arbres et modification du type d'arbre

Ressources




  • Unity 3D

    Nous avons choisi Unity 3D pour notre projet de jeu vidéo car il offre des outils puissants pour créer des environnements immersifs et gérer des assets 3D de manière efficace. Grâce à ses nombreuses fonctionnalités, nous pouvons donner vie à des mécaniques de gameplay dynamiques et à des effets visuels impressionnants.

  • Github / Github Desktop

    Nous avons choisi GitHub pour partager et collaborer sur notre projet, car ils nous permettent de travailler efficacement en équipe. Grâce à cet outil, nous pouvons gérer facilement les versions du code et suivre les modifications de chacun.

  • Blender

    Nous avons utilisé Blender pour créer les maps de notre jeu, car c'est un outil puissant pour concevoir des environnements en 3D avec précision et détail.

  • Photon / Photon Engine

    Nous avons utilisé Photon pour mettre en place le multijoueur de notre jeu, car il offre une solution fiable et performante pour synchroniser les joueurs en temps réel.
